# 2026-07-13-turbopack-native-package-bundling

- Area: `build`
- Context: Deployed the file version history feature (Story 28, PR #118), which introduced direct Turso/`@libsql/client` usage in server actions and API routes. Issue 43 (PR #119) tried `serverExternalPackages` + `createRequire` and still failed in production.
- Symptom: Saves/version history return 500 on Vercel. Logs show `Cannot find package '@libsql/client-bc2a1f2e4d569585'` from `.next/server/chunks/ssr/[turbopack]_runtime.js`.
- Root cause:
  - `@libsql/client` is on Next.js's **default** `serverExternalPackages` list.
  - Turbopack rewrites external packages to hashed aliases (`@libsql/client-<hash>` / `@libsql/client-<hash>/web`) under `.next/node_modules` (Next.js #86375). Those aliases frequently do not survive Vercel serverless packaging (Next.js #93901).
  - `createRequire("@libsql/client")` in app code was insufficient because `drizzle-orm/libsql` still statically imports `@libsql/client`.
  - Switching to `@libsql/client/web` alone still emitted `@libsql/client-<hash>/web` because the package root remains default-externalized.
- Fix:
  - Use fetch-based entries: `@libsql/client/web` + `drizzle-orm/libsql/web`.
  - Add `transpilePackages: ["@libsql/client"]` so Turbopack **bundles** the package instead of emitting a hashed external alias.
  - Do not rely on `serverExternalPackages` / `createRequire` for this package on Vercel — they are the mechanism that creates the broken alias.
- Guardrails:
  - For remote Turso on Vercel/serverless, use `@libsql/client/web` (or `@tursodatabase/serverless`) and pair with `drizzle-orm/libsql/web`.
  - If a package is on Next's default `serverExternalPackages` list and Turbopack hashes it, prefer `transpilePackages` (force bundle) over adding it again to `serverExternalPackages`.
  - After changing DB client imports, grep `.next/server` for `@libsql/client-[a-f0-9]` and confirm `.next/node_modules/@libsql` is absent before treating the build as deploy-safe.
  - Local `pnpm build` success alone is not enough — inspect the emitted server chunks for hashed externals.
